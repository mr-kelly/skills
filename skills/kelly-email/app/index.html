How It Works

/kelly-email reads mail and writes a local batch. This app only edits local review files. After you approve items here, ask /kelly-email to execute approved decisions.

Workflow

  1. Ask /kelly-email to generate a review batch.
  2. Review messages here and approve, request a draft, or leave a note.
  3. Ask /kelly-email to execute approved decisions.

Configuration

Recommended user-wide files:

~/.config/kelly-email/config.yml
~/.config/kelly-email/.env

Alternative local files:

.agents/skills/kelly-email/config.local.yml
.agents/skills/kelly-email/.env.local

The data reader is currently local config. Later it can be swapped for Supabase, Postgres, or pusa-cloud without changing the approval UI contract.

YAML stores accounts, aliases, identities, user profile, brands, official URLs, knowledge sources, style, CTA URLs, and risk keywords. Secrets stay in env files and are referenced by variable name.
